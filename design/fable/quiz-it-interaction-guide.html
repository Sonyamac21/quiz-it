QUIZ-IT · INTERACTION GUIDE

Every user interaction and behaviour across Host, Display and Handset. This is the behavioural contract: engineers implement these flows exactly; anything not specified here follows the Interaction Laws below, and anything not covered by those doesn't ship until it's added here.

GLOBAL INTERACTION LAWS

  • The host's pace is the master clock. No show state auto-advances. Timers expire into a held state (sealed answers) — never into the next state.
  • One primary action per screen, always the most prominent element, always in the same place per surface (handset: bottom bar · host: bottom-right rail).
  • Feedback ≤80ms for every touch, key and click on every surface. Pressed = scale .97 + fill deepen (+Tick haptic on handset).
  • Two-stage commitment for answers (tap select → tap lock). Hold-to-commit for anything irreversible (play card, Stick/Gamble, dump question, end show, remove team, leave team). Holds fill linearly ~900ms; releasing early decays at 2× speed. Accessibility alternative: double-tap-to-commit (handset setting).
  • Destructive host actions: hold only, no keyboard shortcut, confirmation dialog's primary button is the SAFE option.
  • Buttons never move. A control keeps its position across all states of a screen; controls appear/disappear in place, never reflow siblings during play.
  • No scrolling during timed play on the handset. Scrolling exists only in host library/management surfaces and handset settings.
  • Focus: full keyboard traversal on Host (visible double-ring focus). Handset play surfaces are screen-reader traversable in a stable order: question → options (A–D) → lock.

THE QUESTION LIFECYCLE (the product's core loop)

HOST DEAL → Display: card back · Handset: deal-in Pulse, idle → HOST FLIP → Display: question + options + ring starts · Handset: live question → PLAYER TAP option (Tick, others dim, lock bar appears; re-tap another option freely — first change shows speed-bonus warning once) → PLAYER LOCK (vault seal, Lock haptic; answer immutable; handset shows sealed plate) → timer ends OR host CLOSE EARLY → all handsets: "Answers are in. Eyes up." → HOST REVEAL (the Crack on stage; handset verdicts land a half-beat later) → points count-ups → HOST advances: next deal / standings / set-piece.
  • No answer submitted: at reveal the handset shows the correct answer + "No answer this time." Zero-point row persists on the leaderboard.
  • Late lock attempt (tap in flight when timer hits zero): the lock is rejected silently; the sealed state shows "Answers are in." The phone never claims a lock it can't honour.
  • Multi Tap variant: taps toggle tiles (Tick per toggle) until LOCK ALL; per-question timer; Wipeout gates Q6–10 (host toggle) — wrong tap zeroes that question only.
  • Text/number variant: custom numeric pad for numbers; text uses OS keyboard with the show-styled entry; lock = same bar; editing after lock impossible.

HOST INTERACTIONS

The big button

Space bar and click/tap both fire it. Label always names the show consequence. It arms (edge flare) when its moment is ready; when blocked it stays visible with a one-line reason beneath. It is never destructive.

Team drawer

Tap team row → glass drawer. Score stepper applies instantly with a required reason chip (logged with timestamp). Rename applies live everywhere. Remove team = hold. Cancel a played Power Card = hold (dispute tool). Drawer closes on Esc, outside tap, or T — show keeps running beneath it throughout.

Insights & casting

Four cards refresh only at natural pauses (post-reveal, round end) — never mid-question. CAST sends the card to the Display as a caption when the stage is idle; a second CAST replaces the first (one caption max). Cards flare once when their subject changed.

Set-piece consoles

One trigger per act, in a vertical armed sequence; the next trigger lights only when the previous act settles. Fallback controls (SPIN FROM HERE, advance without team input) are holds, present on every set-piece for dead-phone recovery. The finale checklist (song, tie, photo) blocks BEGIN THE ASCENT until each item is confirmed or skipped.

DISPLAY BEHAVIOUR

The Display accepts no direct interaction — it is a pure state renderer driven by the host and game events. Behavioural rules: it may never show two focal elements; it may never render input affordances; every state change arrives through the Breath transition (or a scripted hard cut at Gear 3+); when it loses its feed it degrades through Reconnecting → Connection Lost exactly as specced, holding the last good frame dimmed beneath. On resume it snaps to current state — never replays missed moments.

Eyes Up protocol

Entered automatically at: answers-closed, leaderboard builds, podium, other teams' set-piece moments. The screen drops to near-black (crest + whisper line). Exit is always signalled physically (deal-in Pulse or role ignition Hit) so players can safely ignore the phone. Any tap during Eyes Up shows rank + score for 3s, then returns dark — the phone answers the itch without stealing the stage.

Joining & recovery

NFC tap / QR / URL → PIN keypad (auto-submit on 4th digit) → team name (uniqueness checked live) → crest birth → lobby. Returning device: "Welcome back — rejoin as [team]?" one tap → identity, score, locked answer restored. New device, existing team: same prompt via team name entry match, host-approvable if contested.

Connection behaviour

  • Loss: whisper strip + dim, content stays; ≥10s adds the "show carries on" line; no modal, no red, no haptic on loss.
  • Recovery ≤10s: silent (strip dissolves). Longer: one-line bridge ("Q6 is live, you're 4th") then live state. One Tick.
  • Answers locked pre-drop are honoured; a question missed entirely resolves as no-answer, without comment.

Settings & profile

Reachable from the status strip only outside timed play. Haptics (Full / Reduced / Off), Reduce motion, Text size, Leave team (hold). Profile shows nights played, wins, best superlative, crest geometry picker. Nothing in settings affects fairness.

EDGE CASES & GUARANTEES

  • Tie at the podium: flagged in the finale checklist with the resolution rule stated; never discovered live.
  • Host device dies mid-show: any signed-in device resumes via Session Recovery at the exact phase, timer included.
  • Double-tap race on lock: first lock wins; second tap is a no-op (never an error).
  • Simultaneous Power Cards: resolved in commit order; the Display announces sequentially, 1.5s apart.
  • Team at 0% battery mid-Pursuit: unanswered gates resolve as no-answer; banked points persist; rejoin restores the lane.
  • The room never sees a stack trace, an error code, a spinner, or the word "error." Ever.
