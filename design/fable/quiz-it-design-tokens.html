ICONOGRAPHY & COMPONENT SIZING

Geometric line icons, 24×24 grid, 2px stroke (1.5 @16 / 3 @40+), round caps, no fills, no two-tone, no emoji in product UI. Host icons always labelled. Display uses ▲▼ + card sigils only. Sizes: --ic-s 16 · --ic-m 24 · --ic-l 40. Core component sizes: handset option 64h · lock bar 56h · monolith 96h · host row 44h · display leaderboard row 96h · crest 18–52px by context · Power Card aspect 2:3 · playing card aspect 2:3.

ACCESSIBILITY (BINDING)

Contrast: Display ≥7:1, Handset/Host ≥4.5:1 (whisper ≥3:1). Meaning never by colour alone — monochrome test every state. Fully playable muted. ≤3 flashes/sec; light-shocks single, ≥250ms apart. Reduce-motion: entrances→fades, travels→dissolves, particles→static light, count-ups→instant+pulse. Haptics always paired with visuals; reduce-haptics keeps Lock + reveal Hit. Hold-to-commit offers double-tap alternative. Timers: clock never starts before content lands; host +15s always available; nothing but answering has a timeout.

INTERACTION LAWS (BINDING)

One primary action per screen. Two-stage answering (tap select → lock commit). Hold-to-commit for anything irreversible; charging is mirrored where the room should see hesitation. Destructive host actions: hold, no keyboard shortcut, safe option is primary. Buttons never swap positions between states. Feedback ≤80ms for every touch. The host's pace is the master clock — nothing auto-advances a show beat. Full interaction flows: see the Interaction Guide.
