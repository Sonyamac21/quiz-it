# PURPOSE

This folder contains the official product specifications for Quiz-It.

These documents define the product.

They are NOT suggestions.

They are the authoritative source for all future engineering work.

Nothing should be redesigned unless these documents are formally updated.

---

# PROJECT VISION

Quiz-It is not quiz software.

Quiz-It is a premium hosted live entertainment platform.

The software exists to support the host and entertain the room.

Everything must reinforce three core principles:

Display = The Theatre

Host = Mission Control

Handset = The Controller

---

# ORDER OF PRECEDENCE

If two documents ever conflict, follow them in this order:

1. Product Constitution
2. Experience Bible
3. Visual Design System
4. Motion Language
5. Audio Style Guide
6. Brand Voice Guide
7. Display Storyboards
8. Live Show Specification
9. Host Mission Control
10. Player Handset Specification
11. Pre-Show Experience
12. Venue Operations Manual
13. Accessibility Standard

If a conflict still exists:

STOP.

Do not guess.

Explain the conflict before implementing anything.

---

# ENGINEERING RULES

Never redesign.

Never reinterpret.

Never "improve" the specifications.

Implement exactly what the specifications describe.

If implementation becomes impossible because of technical limitations, explain the limitation before changing the design.

No speculative improvements.

No feature creep.

---

# DEVELOPMENT WORKFLOW

Every task follows the same order.

1. Select one specification.

2. Implement only that specification.

3. Run the build.

4. Test the feature.

5. Commit.

6. Push.

7. Move to the next task.

Never work on multiple unrelated features simultaneously.

# PRODUCT RULES

The host is always the star.

The Display creates the theatre.

The handset supports the player.

Motion always has purpose.

Audio never competes with the host.

No scrolling during gameplay.

Everything important is understood within one second.

The room is one audience.

Every celebration belongs to a tension level.

The software supports the performance.

The host controls the performance.
